12 The causal question

We use a two-time-point HIV cohort to make the mechanics visible. The question: what is the probability of virologic failure if every patient maintained high ART adherence at each visit, compared to never maintaining high adherence?

This is a comparison of two static regimes, \(\bar{a} = (1,1)\) versus \(\bar{a} = (0,0)\), with treatment-confounder feedback through CD4 count. We use two time points for clarity; the same logic extends to any number of visits.

Show code 
dag_hiv <- dagify(
  A1 ~ W, L1 ~ A1 + W, A2 ~ L1 + A1 + W, Y ~ A1 + A2 + L1 + W,
  exposure = "A1", outcome = "Y",
  labels = c(W = "Baseline\n(age, sex, CD4_0, VL_0)",
             A1 = "ART adherence\nvisit 1",
             L1 = "CD4 count\nvisit 1",
             A2 = "ART adherence\nvisit 2",
             Y = "Virologic\nfailure"),
  coords = list(
    x = c(W = 0, A1 = 1, L1 = 2, A2 = 3, Y = 4),
    y = c(W = 1, A1 = 0, L1 = 0.8, A2 = 0, Y = 0)
  )
)
ggdag(dag_hiv, text = FALSE, use_labels = "label", text_size = 2.8) +
  theme_dag()

13 Simulated data

Show code 
set.seed(2026)
n <- 2000

W_age    <- rnorm(n, mean = 40, sd = 10)
W_male   <- rbinom(n, 1, 0.6)
W_cd4_bl <- pmax(50, rnorm(n, mean = 350, sd = 120))
W_vl_bl  <- rnorm(n, mean = 4.2, sd = 0.7)
W <- 0.01 * (W_age - 40) - 0.001 * (W_cd4_bl - 350) + 0.2 * W_vl_bl - 0.1 * W_male

A1 <- rbinom(n, 1, plogis(0.3 + 0.5 * W))
L1 <- rnorm(n, mean = 0.3 * W + 1.2 * A1, sd = 1)  # strong treatment -> confounder
A2 <- rbinom(n, 1, plogis(-0.5 + 0.8 * L1 + 0.3 * A1))  # strong confounder -> treatment
Y  <- rbinom(n, 1, plogis(-0.8 - 0.4 * A1 - 0.5 * A2 - 0.6 * L1 + 0.2 * W))  # strong L1 effect

dat <- tibble(W, A1, L1, A2, Y)
cat("Failure rate:", round(mean(Y), 3), "\n")
#> Failure rate: 0.192

14 The backwards regression idea

L-TMLE estimates the causal effect through three ideas working together.

Sequential regressions, working backwards. Start at the final outcome and model \(E[Y \mid A_2, L_1, A_1, W]\). This conditions on \(L_1\) to deconfound \(A_2\). Then move backwards: take the predicted outcomes under \(A_2 = 1\) and regress them on \((A_1, W)\) only, without conditioning on \(L_1\). This integrates \(L_1\) out, preserving the mediated effect of \(A_1\) through \(L_1\).

Treatment mechanism models. Estimate the probability of receiving treatment at each visit conditional on history: \(\hat{g}_1(W)\) and \(\hat{g}_2(A_1, L_1, W)\).

Targeting steps. Use the treatment mechanism to construct clever covariates that correct the sequential regressions for any remaining bias. At each time point, the final estimate is consistent if either the outcome regression or the treatment model is correctly specified (sequential double robustness).

The key insight: conditioning on \(L_1\) at time 2 deconfounds \(A_2\), while NOT conditioning on \(L_1\) at time 1 preserves the indirect effect \(A_1 \to L_1 \to Y\). Standard regression cannot do both in a single model.

15 How the recipe works in plain language

Step 1. At each visit, model how likely each patient was to actually adhere. Sicker patients are less likely to adhere; patients who already missed doses are less likely to catch up.

Step 2. Starting at visit 2, predict each patient’s failure risk as if they had high adherence. Adjust using the treatment model so that the prediction is calibrated for the causal estimand, not just for prediction accuracy. This is the targeting step.

Step 3. Take the adjusted visit-2 predictions and use them as the “outcome” for visit 1. Predict under high adherence at visit 1, and target again. The visit-1 model does not condition on CD4, because CD4 is a consequence of visit-1 adherence.

Step 4. Average the final targeted predictions across all patients. This is the L-TMLE estimate of \(E[Y(1,1)]\).

16 Naive regression vs L-TMLE

A quick demonstration of why the backwards structure matters:

Show code 
# Oracle truth
set.seed(9999)
n_mc <- 100000; W_mc <- rep(W, length.out = n_mc)
L1_cf11 <- rnorm(n_mc, mean = 0.3 * W_mc + 1.2, sd = 1)
true_EY_11 <- mean(plogis(-0.8 - 0.4 - 0.5 - 0.6 * L1_cf11 + 0.2 * W_mc))

# Naive: condition on L1 (blocks A1 -> L1 -> Y)
naive_mod <- glm(Y ~ A1 + A2 + L1 + W, family = binomial, data = dat)
naive_pred <- mean(predict(naive_mod, newdata = dat %>% mutate(A1 = 1, A2 = 1), type = "response"))

cat("True E[Y(1,1)]:", round(true_EY_11, 4), "\n")
#> True E[Y(1,1)]: 0.0941
cat("Naive (conditions on L1):", round(naive_pred, 4), "\n")
#> Naive (conditions on L1): 0.1411

The naive estimate is biased because conditioning on \(L_1\) blocks part of the causal pathway. L-TMLE avoids this by conditioning on \(L_1\) only where it confounds (\(A_2\)) and integrating it out where it mediates (\(A_1\)).

17 When static regimes break down

The “always adhere” regime is pedagogically useful, but it has a practical limitation: no real intervention achieves 100% adherence. When the static regime assigns treatment values that are rare in the observed data, positivity breaks down: propensity scores approach zero, clever covariates explode, and the estimate becomes unstable.

This is not a failure of L-TMLE as a method. It is a limitation of the estimand. Asking “what if everyone achieved perfect adherence?” requires extrapolating far outside the data for patients who are very unlikely to adhere. The result is high variance and potential bias.

The solution is to define more realistic interventions that stay within the observed data support. The LMTP framework provides this: instead of forcing a specific treatment value, it shifts the treatment process by a feasible amount. The LMTP chapters develop this approach.

18 Package implementation

The ltmle R package automates the full L-TMLE procedure. For static regimes, the interface is straightforward:

Show code 
library(ltmle)

dat_ltmle <- data.frame(W = dat$W, A1 = dat$A1, L1 = dat$L1, A2 = dat$A2, Y = dat$Y)

result <- ltmle(
  data = dat_ltmle,
  Anodes = c("A1", "A2"),
  Lnodes = c("L1"),
  Ynodes = "Y",
  abar = list(treatment = c(1, 1), control = c(0, 0)),
  SL.library = c("SL.glm", "SL.mean")
)

summary(result)

For realistic interventions estimated with lmtp, see the LMTP case study and the applied workflow chapter.

18.1 Check your understanding

Q1. Why does L-TMLE work backwards through time?

Working backwards allows L-TMLE to condition on \(L_t\) where it confounds future treatment and integrate it out where it mediates past treatment. A forward approach cannot do both simultaneously.

Q2. What is sequential double robustness?

At each time point, L-TMLE is consistent if either the outcome model or the treatment model is correctly specified. Different models can fail at different time points, as long as at each point at least one is correct.

Q3. When should you use LMTP instead of static L-TMLE?

When the static regime (e.g., “always adhere”) assigns treatment values that are rare in the observed data. LMTP defines feasible shifts that stay within the data support, producing more stable estimates and answering more clinically relevant questions.
