8 Chapter 2.3: Doubly Robust Estimation and Targeted Learning

Bridging outcome modeling and weighting for more robust causal effect estimation

In previous chapters, you learned:

  • G-computation depends on correctly modeling the outcome
  • IPTW depends on correctly modeling the treatment mechanism

G-computation depends entirely on the outcome model; IPTW depends entirely on the treatment model. AIPW and TMLE combine both, providing consistency if either model is correctly specified.

9 Why Doubly Robust Estimators?

Key Idea: Why Doubly Robust Estimators?

A doubly robust estimator provides consistency if either the treatment model or the outcome model is correctly specified (Laan and Rose 2011). By combining both sources of information, doubly robust estimators provide two chances to get it right and often outperform g-computation or IPTW used alone. The goal remains the same as in previous chapters: estimate \(E[Y(1)] - E[Y(0)]\), the population-level risk difference.

10 AIPW: Augmented Inverse Probability Weighting

What Makes AIPW “Doubly Robust”?

AIPW adds a correction term to IPTW using the outcome regression. The key insight is that it is constructed so that:

  • If the treatment model is correctly specified (and positivity holds), the IPTW component yields a consistent estimate of the causal effect
  • If the outcome model is correctly specified, the augmentation (G-computation) component corrects any remaining bias from the weights
  • If both are correctly specified, the estimator achieves the semiparametric efficiency bound, meaning it has the smallest possible asymptotic variance among all regular asymptotically linear estimators

In other words, AIPW “augments” the weighted estimator with a bias-correction term derived from the outcome model. This makes the estimator consistent under correct specification of either (but not necessarily both) nuisance models, plus positivity; hence the name doubly robust.

Note that “doubly robust” is a consistency property: it guarantees the estimator converges to the truth as \(n \to \infty\) under the stated conditions, not that it is unbiased in finite samples.

10.1 Formula

Let:

  • \(g(W) = P(A = 1 \mid W)\) be the propensity score
  • \(m(a, W) = E[Y \mid A = a, W]\) be the outcome regression

Then:

\[ \hat{\mu}_1 = \frac{1}{n} \sum_i \left[ m(1, W_i) + \frac{A_i}{g(W_i)} \big(Y_i - m(1, W_i)\big) \right] \]

\[ \hat{\mu}_0 = \frac{1}{n} \sum_i \left[ m(0, W_i) + \frac{1 - A_i}{1 - g(W_i)} \big(Y_i - m(0, W_i)\big) \right] \]

ATE = \(\hat{\mu}_1 - \hat{\mu}_0\)

Each formula has the same structure: start with the outcome model prediction \(m(a, W_i)\), then add an IPW-weighted residual that corrects for any bias in the outcome model. If \(m\) is correct, the residuals average to zero and the correction vanishes. If \(m\) is wrong but \(g\) is correct, the IPW correction fixes the bias.

11 Simulated Osteoporosis Dataset

library(tidyverse)

set.seed(2026)
n <- 4000

age <- rnorm(n, 75, 6)
cvd <- rbinom(n, 1, plogis(0.12 * (age - 70)))

# Treatment model
A <- rbinom(n, 1, plogis(-1 + 0.08*(age - 70) + 1.4*cvd))

# Outcome model
Y <- rbinom(n, 1, plogis(-2 + 0.6*A + 0.10*(age - 70) + 1.0*cvd))

dat <- tibble(age, cvd, A, Y)

12 Fit Outcome and Propensity Score Models

Qmod <- glm(Y ~ A + age + cvd, family = binomial, data = dat)
gmod <- glm(A ~ age + cvd, family = binomial, data = dat)

dat <- dat %>%
  mutate(
    ps = predict(gmod, type = "response"),
    Q1 = predict(Qmod, newdata = mutate(dat, A=1), type = "response"),
    Q0 = predict(Qmod, newdata = mutate(dat, A=0), type = "response")
  )

13 Compute AIPW Estimator

with(dat, {
  mu1 <- mean( Q1 + A * (Y - Q1) / ps )
  mu0 <- mean( Q0 + (1 - A) * (Y - Q0) / (1 - ps) )
  ate <- mu1 - mu0
  c(mu1 = mu1, mu0 = mu0, ate = ate)
})
Interpreting the AIPW Estimate

Each AIPW component starts with the outcome-model prediction and adds a weighted residual correction. For instance, Q1 + A * (Y - Q1) / ps corrects \(m(1, W)\) by upweighting the residual \((Y - Q1)\) for treated individuals. This structure gives AIPW its doubly robust property: even if \(Q\) is wrong, the IPW correction fixes the bias (and vice versa).

14 TMLE: Targeted Maximum Likelihood Estimation

Key Idea: Why TMLE Over AIPW?

TMLE (Laan and Rubin 2006) offers several advantages over AIPW: predicted probabilities always stay in \([0, 1]\) (AIPW can fall outside), it integrates naturally with ensemble machine learning (Super Learner), and the fluctuation step solves the efficient influence curve equation by construction. This provides a foundation for valid inference under regularity conditions: correct specification of at least one nuisance model, positivity, and (when data-adaptive learners are used) cross-fitting or CV-TMLE to avoid Donsker-class requirements (see Zheng and Laan (2011)).

15 TMLE Step-by-Step

Step 1: Fit the initial outcome model (Q). Fit a model for \(E[Y \mid A, W]\) and generate initial predictions under both treatment values for each individual: \(Q_n^0(1, W_i)\) (predicted outcome if treated) and \(Q_n^0(0, W_i)\) (predicted outcome if untreated). These predictions are “initial” because they have not yet been targeted toward the causal parameter of interest.

logit <- function(p) log(p/(1-p))
expit <- function(x) 1/(1+exp(-x))

# Step 1: Initial Q predictions at observed A
Qinit <- predict(Qmod, type="response")

Step 2: Estimate the propensity score. The propensity score \(g(W) = P(A = 1 \mid W)\) is needed to construct the clever covariate in the next step. We already fit this model in Section 4.

# Step 2: Propensity scores (already estimated)
ps <- dat$ps
Definition: The Clever Covariate

The clever covariate \(H(A, W)\) is the key ingredient that makes TMLE “targeted.” It is derived from the efficient influence curve for the ATE:

\[ H(A, W) = \frac{A}{g(W)} - \frac{1 - A}{1 - g(W)} \]

For computing the counterfactual means, we also need the treatment-specific components:

  • \(H_1(W) = \frac{1}{g(W)}\) (for the treated potential outcome)
  • \(H_0(W) = \frac{-1}{1 - g(W)}\) (for the control potential outcome)

The clever covariate directs the fluctuation to reduce bias for the specific causal estimand we care about, the ATE.

# Step 3: Clever covariate at observed treatment
H <- with(dat, A/ps - (1-A)/(1-ps))
1
\(H(A,W)\) is derived from the efficient influence curve for the ATE. For treated observations (\(A=1\)) it equals \(1/g(W)\); for controls (\(A=0\)) it equals \(-1/(1-g(W))\). This combined form simultaneously handles both arms and directs the fluctuation step specifically toward the ATE estimand rather than any other parameter.
Step 4: Fit the Fluctuation Model and Update Predictions

This is the critical “targeting” step. We fit a logistic regression of \(Y\) on \(H\) with the initial logit-predictions as an offset:

\[ \text{logit}(E[Y \mid A, W]) = \text{logit}(Q_n^0(A, W)) + \epsilon \cdot H(A, W) \]

The estimated \(\hat{\epsilon}\) is then used to update (fluctuate) the initial predictions. Crucially, we must apply this fluctuation to the counterfactual predictions \(Q_n^0(1, W)\) and \(Q_n^0(0, W)\) using their respective clever covariates \(H_1\) and \(H_0\).

This step ensures the final estimates solve the efficient influence curve equation, which is what gives TMLE its optimal statistical properties.

The targeting step is not trying to improve prediction in general. It is adjusting the initial outcome predictions specifically to reduce bias for the ATE. A well-calibrated prediction model might predict individual outcomes accurately but still produce a biased ATE when predictions are averaged. The fluctuation step corrects for exactly this discrepancy by incorporating information from the propensity score.

# Step 4: Fluctuation model
epsilon <- glm(dat$Y ~ -1 + offset(logit(Qinit)) + H,
               family = binomial)$coef
1
The -1 suppresses the intercept; offset(logit(Qinit)) fixes the initial Q predictions on the logit scale so only the coefficient on H (the fluctuation parameter ε) is estimated
2
Logistic family keeps the update on the probability scale; .coef extracts the single ε; a small number close to zero means the initial Q needed little correction

16 Counterfactual Predictions for TMLE

# Get initial (pre-fluctuation) predictions under each treatment
Q1_init <- predict(Qmod, newdata = mutate(dat, A=1), type="response")
Q0_init <- predict(Qmod, newdata = mutate(dat, A=0), type="response")

# Clever covariates for counterfactual treatment values
H1 <- 1 / ps
H0 <- -1 / (1 - ps)

# Apply the fluctuation to counterfactual predictions
Q1_star <- plogis(logit(Q1_init) + epsilon * H1)
Q0_star <- plogis(logit(Q0_init) + epsilon * H0)

# TMLE ATE estimate
tmle_ate <- mean(Q1_star) - mean(Q0_star)
tmle_ate
1
Treatment-arm-specific clever covariates: \(H_1(W) = 1/g(W)\) targets the treated potential outcome; \(H_0(W) = -1/(1-g(W))\) targets the control potential outcome. These must be evaluated at the counterfactual treatment value, not the observed one.
2
Apply the fluctuation on the logit scale, then plogis() maps back to probability; guaranteeing all targeted predictions stay in \([0,1]\), unlike AIPW which can violate these bounds
3
The TMLE substitution estimator: simply take the sample mean of each arm’s targeted predictions. No explicit weighting needed; the clever covariate already incorporated the propensity score.

The initial outcome and propensity score models do not have to be simple GLMs. They can be estimated with Super Learner, an ensemble machine learning approach covered in the next chapter, which reduces sensitivity to model misspecification.

17 TMLE Inference: Standard Errors and Confidence Intervals

A major advantage of TMLE is that inference follows directly from the efficient influence curve (EIC). Using the clever covariate notation from Section 7, where \(H(A, W) = A/g(W) - (1-A)/(1-g(W))\), the EIC for the ATE at the TMLE fit can be written in a single unified form:

\[ D_i = H(A_i, W_i)\big(Y_i - Q^*(A_i, W_i)\big) + Q^*(1, W_i) - Q^*(0, W_i) - \hat{\psi} \]

Equivalently, expanding \(H\) into its treatment-arm-specific components \(H_1(W) = 1/g(W)\) and \(H_0(W) = -1/(1-g(W))\) (which already include the sign for each arm):

\[ D_i = \frac{A_i}{e(W_i)}\big(Y_i - Q_1^*(W_i)\big) - \frac{1-A_i}{1-e(W_i)}\big(Y_i - Q_0^*(W_i)\big) + Q_1^*(W_i) - Q_0^*(W_i) - \hat{\psi} \]

where \(\hat{\psi}\) is the TMLE ATE estimate. The variance of the ATE is estimated by the sample variance of the EIC divided by \(n\): \(\hat{\sigma}^2 = \frac{1}{n}\sum_i D_i^2\).

When is EIC-based inference valid?

The EIC-based variance estimator and resulting Wald-type confidence intervals are asymptotically valid when (i) at least one nuisance model is consistently estimated, (ii) positivity holds, and (iii) certain regularity/rate conditions are met. When data-adaptive or machine learning methods are used for nuisance estimation, these conditions require cross-fitting (sample-splitting) or CV-TMLE to avoid empirical process conditions (Donsker class membership) that flexible learners typically violate. In the tmle3 and drtmle R packages, cross-fitting is available; the classic tmle package does not cross-fit by default but remains valid when the Super Learner library is not too aggressive (e.g., excludes highly adaptive learners like HAL without cross-fitting). See Zheng and Laan (2011) for the theoretical foundations.

# Observed outcomes
Y_obs <- dat$Y

# EIC (efficient influence curve) components
# For treated: use Q1_star; for control: use Q0_star
# But we need Q* at the OBSERVED A for the residual terms
Qstar_obs <- ifelse(dat$A == 1, Q1_star, Q0_star)
H_obs <- ifelse(dat$A == 1, H1, H0)

# Efficient influence curve for each observation
EIC <- H_obs * (Y_obs - Qstar_obs) + Q1_star - Q0_star - tmle_ate

# Standard error and 95% CI
tmle_se <- sqrt(var(EIC) / n)
ci_lower <- tmle_ate - 1.96 * tmle_se
ci_upper <- tmle_ate + 1.96 * tmle_se

tibble(
  Estimate = tmle_ate,
  SE = tmle_se,
  CI_lower = ci_lower,
  CI_upper = ci_upper
)
1
Select the targeted Q prediction and clever covariate at each observation’s observed treatment value; needed to form the IPW-weighted residual term that corrects for outcome model error
2
The EIC has two parts: (a) \(H(A,W)(Y - Q^*)\); an IPW-weighted residual that corrects any outcome model bias, and (b) \((Q_1^* - Q_0^* - \hat\psi)\); the centered substitution component. When the fluctuation step correctly solved the EIC equation, the sample mean of EIC is exactly zero.
3
The EIC variance divided by \(n\) yields the semiparametric efficient variance; this is valid even when machine learning is used for nuisance models (with appropriate cross-fitting)
Interpreting TMLE Inference

The EIC-based standard error is a key output of TMLE. Because the fluctuation step solves the EIC estimating equation, the resulting confidence interval achieves approximately correct coverage under regularity conditions, even when machine learning is used for the nuisance models (provided cross-fitting or other sample-splitting techniques are employed).

If the confidence interval excludes zero, we have evidence that the treatment has a non-null causal effect on the outcome at the 95% confidence level.

18 Comparison: AIPW vs TMLE

18.0.1 AIPW vs TMLE: Head-to-Head

Property AIPW TMLE
Doubly robust Yes Yes
Semiparametrically efficient Yes (asymptotically) Yes
Handles ML well Yes, with cross-fitting Yes, with SuperLearner + cross-fitting (CV-TMLE)
Predictions bounded in \([0,1]\) No Yes
Solves EIC equation Not guaranteed Yes, by construction
Implementation complexity Simple Moderate

Key point. Both AIPW and TMLE are valid doubly robust estimators. TMLE is generally preferred because it respects the bounds of the parameter space (e.g., probabilities stay in \([0,1]\)), integrates naturally with ensemble machine learning, and solves the efficient influence curve equation by construction. In practice, this means TMLE tends to have better finite-sample performance and more trustworthy inference.

19 Running Example: Antiretroviral Therapy Cohort

See the Running Case Study for the full data description.

In this section we apply TMLE to the HIV antiretroviral therapy cohort that serves as the running example throughout this handbook. The goal is to estimate the causal effect of ever initiating HAART (ever_haart) on mortality (died), adjusting for baseline covariates age, sex, and cd4_sqrt_baseline.

TMLE is a natural fit for this dataset because the relationship between CD4 count, treatment initiation, and mortality is complex. Having two chances to get the model right (through the outcome regression and the propensity score) is especially valuable when confounding by indication is strong: patients with lower CD4 counts are more likely to initiate HAART and also more likely to die.

This is a point-treatment analysis that collapses each patient’s longitudinal record into a single baseline snapshot. The original panel data contain time-varying CD4 measurements and treatment switches, but here we simplify to a single treatment indicator (ever_haart) and a single outcome (died). Longitudinal TMLE, which accounts for time-varying treatment and confounding in the full panel data, is covered in Chapter 3.9.

Interpreting the HAART TMLE Result

A positive ATE would suggest that HAART initiation increases the probability of death in this simple point-treatment model. However, this likely reflects confounding by indication: patients with more advanced disease (lower CD4 counts) are both more likely to initiate treatment and more likely to die. The baseline snapshot cannot fully capture the time-varying nature of CD4 decline, treatment initiation, and mortality. Chapter 3.9 addresses this with longitudinal TMLE and marginal structural models that properly handle time-varying confounding.

20 Summary

Key Takeaways
  • Doubly robust estimators (AIPW, TMLE) combine outcome and treatment models and are consistent if either model is correct
  • TMLE is preferred: bounded predictions, plug-in estimation, natural ML integration, and EIC-based inference
  • Doubly robust estimators are the recommended default for point-treatment causal inference in modern epidemiology

20.1 Check your understanding

Q1. What does “doubly robust” mean in practice? Does it mean you can ignore model specification entirely?

No. “Doubly robust” means the estimator is consistent if either the treatment model or the outcome model is correctly specified, but not necessarily if both are wrong. It does not give you license to ignore model specification. If both nuisance models are badly misspecified, the estimator can be biased. In practice, you should still try to specify both models as carefully as possible. The double robustness property is an insurance policy, not a substitute for thoughtful modeling.

Q2. Why does TMLE use a “clever covariate” derived from the propensity score, and what happens if propensity scores are extreme?

The clever covariate \(H(A, W) = A / g(W) - (1 - A) / (1 - g(W))\) is derived from the efficient influence curve for the ATE. It directs the fluctuation of the initial outcome model toward the specific causal parameter of interest, ensuring that the final estimate solves the EIC estimating equation. If propensity scores are extreme (near 0 or 1), the clever covariate becomes very large, which can destabilize the fluctuation step. This is why practical positivity (adequate overlap in propensity score distributions) is important for TMLE just as it is for IPTW.

Q3. Your TMLE estimate of a risk probability is 0.85. Could AIPW have produced an estimate above 1.0 for the same data?

Yes. AIPW is a linear estimator that does not enforce the bounds of the parameter space. Because it adds a weighted residual correction to the outcome-model prediction, the resulting estimate can fall outside \([0, 1]\), especially when propensity scores are extreme and the correction term is large. TMLE avoids this by performing its update on the logistic (log-odds) scale and then back-transforming, which guarantees that predicted probabilities remain in \([0, 1]\). This is one of the key practical advantages of TMLE over AIPW.

20.3 Software Implementation (R)

This example demonstrates the double robustness property using the tmle package. We deliberately misspecify one nuisance model at a time to show that the TMLE estimate remains consistent as long as at least one model is correct.

  • Simulate data with a nonlinear confounder–outcome relationship
  • Scenario A: correct \(Q\), wrong \(g\) → TMLE consistent
  • Scenario B: wrong \(Q\), correct \(g\) → TMLE consistent
  • Scenario C: both correct → TMLE efficient
  • Falls back to a manual demonstration if tmle is unavailable

Y <- 0.5 * A + sin(W) + rnorm(n, sd = 0.3)  # true ATE = 0.5

# Chapter 2.3: Doubly Robust Estimation and Targeted Learning
*Bridging outcome modeling and weighting for more robust causal effect estimation*

In previous chapters, you learned:

- **G-computation** depends on correctly modeling the *outcome*
- **IPTW** depends on correctly modeling the *treatment mechanism*

G-computation depends entirely on the outcome model; IPTW depends entirely on the treatment model. AIPW and TMLE combine both, providing consistency if **either** model is correctly specified.

---

# Why Doubly Robust Estimators?

:::{.callout-tip}
## Key Idea: Why Doubly Robust Estimators?

A doubly robust estimator provides consistency if **either** the treatment model or the outcome model is correctly specified [@vanderLaan2011targeted]. By combining both sources of information, doubly robust estimators provide two chances to get it right and often outperform g-computation or IPTW used alone. The goal remains the same as in previous chapters: estimate $E[Y(1)] - E[Y(0)]$, the population-level risk difference.
:::

---

# AIPW: Augmented Inverse Probability Weighting

:::{.callout-note}
## What Makes AIPW "Doubly Robust"?

AIPW adds a correction term to IPTW using the outcome regression. The key insight is that it is constructed so that:

- If the **treatment model** is correctly specified (and positivity holds), the IPTW component yields a consistent estimate of the causal effect
- If the **outcome model** is correctly specified, the augmentation (G-computation) component corrects any remaining bias from the weights
- If **both** are correctly specified, the estimator achieves the **semiparametric efficiency bound**, meaning it has the smallest possible asymptotic variance among all regular asymptotically linear estimators

In other words, AIPW "augments" the weighted estimator with a bias-correction term derived from the outcome model. This makes the estimator consistent under correct specification of either (but not necessarily both) nuisance models, plus positivity; hence the name *doubly robust*.

Note that "doubly robust" is a *consistency* property: it guarantees the estimator converges to the truth as $n \to \infty$ under the stated conditions, not that it is unbiased in finite samples.
:::

## Formula

Let:

- $g(W) = P(A = 1 \mid W)$ be the propensity score
- $m(a, W) = E[Y \mid A = a, W]$ be the outcome regression

Then:

$$
\hat{\mu}_1 = \frac{1}{n} \sum_i \left[ m(1, W_i) + \frac{A_i}{g(W_i)} \big(Y_i - m(1, W_i)\big) \right]
$$

$$
\hat{\mu}_0 = \frac{1}{n} \sum_i \left[ m(0, W_i) + \frac{1 - A_i}{1 - g(W_i)} \big(Y_i - m(0, W_i)\big) \right]
$$

ATE = $\hat{\mu}_1 - \hat{\mu}_0$

Each formula has the same structure: start with the outcome model prediction $m(a, W_i)$, then add an IPW-weighted residual that corrects for any bias in the outcome model. If $m$ is correct, the residuals average to zero and the correction vanishes. If $m$ is wrong but $g$ is correct, the IPW correction fixes the bias.

# Simulated Osteoporosis Dataset

```{webr-r}
library(tidyverse)

set.seed(2026)
n <- 4000

age <- rnorm(n, 75, 6)
cvd <- rbinom(n, 1, plogis(0.12 * (age - 70)))

# Treatment model
A <- rbinom(n, 1, plogis(-1 + 0.08*(age - 70) + 1.4*cvd))

# Outcome model
Y <- rbinom(n, 1, plogis(-2 + 0.6*A + 0.10*(age - 70) + 1.0*cvd))

dat <- tibble(age, cvd, A, Y)
```

---

# Fit Outcome and Propensity Score Models

```{webr-r}
Qmod <- glm(Y ~ A + age + cvd, family = binomial, data = dat)
gmod <- glm(A ~ age + cvd, family = binomial, data = dat)

dat <- dat %>%
  mutate(
    ps = predict(gmod, type = "response"),
    Q1 = predict(Qmod, newdata = mutate(dat, A=1), type = "response"),
    Q0 = predict(Qmod, newdata = mutate(dat, A=0), type = "response")
  )
```

---

# Compute AIPW Estimator

```{webr-r}
with(dat, {
  mu1 <- mean( Q1 + A * (Y - Q1) / ps )
  mu0 <- mean( Q0 + (1 - A) * (Y - Q0) / (1 - ps) )
  ate <- mu1 - mu0
  c(mu1 = mu1, mu0 = mu0, ate = ate)
})
```

:::{.callout-tip}
## Interpreting the AIPW Estimate

Each AIPW component starts with the outcome-model prediction and adds a weighted residual correction. For instance, `Q1 + A * (Y - Q1) / ps` corrects $m(1, W)$ by upweighting the residual $(Y - Q1)$ for treated individuals. This structure gives AIPW its doubly robust property: even if $Q$ is wrong, the IPW correction fixes the bias (and vice versa).
:::

# TMLE: Targeted Maximum Likelihood Estimation

:::{.callout-tip}
## Key Idea: Why TMLE Over AIPW?

TMLE [@vanderLaan2006tmle] offers several advantages over AIPW: predicted probabilities always stay in $[0, 1]$ (AIPW can fall outside), it integrates naturally with ensemble machine learning (Super Learner), and the fluctuation step solves the efficient influence curve equation by construction. This provides a foundation for valid inference under regularity conditions: correct specification of at least one nuisance model, positivity, and (when data-adaptive learners are used) cross-fitting or CV-TMLE to avoid Donsker-class requirements (see @zheng2011crossvalidated).
:::

---

# TMLE Step-by-Step

**Step 1: Fit the initial outcome model (Q).** Fit a model for $E[Y \mid A, W]$ and generate initial predictions under both treatment values for each individual: $Q_n^0(1, W_i)$ (predicted outcome if treated) and $Q_n^0(0, W_i)$ (predicted outcome if untreated). These predictions are "initial" because they have not yet been targeted toward the causal parameter of interest.

```{webr-r}
logit <- function(p) log(p/(1-p))
expit <- function(x) 1/(1+exp(-x))

# Step 1: Initial Q predictions at observed A
Qinit <- predict(Qmod, type="response")
```

**Step 2: Estimate the propensity score.** The propensity score $g(W) = P(A = 1 \mid W)$ is needed to construct the clever covariate in the next step. We already fit this model in Section 4.

```{webr-r}
# Step 2: Propensity scores (already estimated)
ps <- dat$ps
```

:::{.callout-note}
## Definition: The Clever Covariate

The **clever covariate** $H(A, W)$ is the key ingredient that makes TMLE "targeted." It is derived from the efficient influence curve for the ATE:

$$
H(A, W) = \frac{A}{g(W)} - \frac{1 - A}{1 - g(W)}
$$

For computing the counterfactual means, we also need the treatment-specific components:

- $H_1(W) = \frac{1}{g(W)}$ (for the treated potential outcome)
- $H_0(W) = \frac{-1}{1 - g(W)}$ (for the control potential outcome)

The clever covariate directs the fluctuation to reduce bias for the specific causal estimand we care about, the ATE.
:::

```{webr-r}
# Step 3: Clever covariate at observed treatment
H <- with(dat, A/ps - (1-A)/(1-ps))  # <1>
```
1. $H(A,W)$ is derived from the efficient influence curve for the ATE. For treated observations ($A=1$) it equals $1/g(W)$; for controls ($A=0$) it equals $-1/(1-g(W))$. This combined form simultaneously handles both arms and directs the fluctuation step specifically toward the ATE estimand rather than any other parameter.

:::{.callout-important}
## Step 4: Fit the Fluctuation Model and Update Predictions

This is the critical "targeting" step. We fit a logistic regression of $Y$ on $H$ with the initial logit-predictions as an offset:

$$
\text{logit}(E[Y \mid A, W]) = \text{logit}(Q_n^0(A, W)) + \epsilon \cdot H(A, W)
$$

The estimated $\hat{\epsilon}$ is then used to **update** (fluctuate) the initial predictions. Crucially, we must apply this fluctuation to the counterfactual predictions $Q_n^0(1, W)$ and $Q_n^0(0, W)$ using their respective clever covariates $H_1$ and $H_0$.

This step ensures the final estimates solve the efficient influence curve equation, which is what gives TMLE its optimal statistical properties.

The targeting step is not trying to improve prediction in general. It is adjusting the initial outcome predictions specifically to reduce bias for the ATE. A well-calibrated prediction model might predict individual outcomes accurately but still produce a biased ATE when predictions are averaged. The fluctuation step corrects for exactly this discrepancy by incorporating information from the propensity score.
:::

```{webr-r}
# Step 4: Fluctuation model
epsilon <- glm(dat$Y ~ -1 + offset(logit(Qinit)) + H,  # <1>
               family = binomial)$coef                   # <2>
```
1. The `-1` suppresses the intercept; `offset(logit(Qinit))` fixes the initial Q predictions on the logit scale so only the coefficient on `H` (the fluctuation parameter ε) is estimated
2. Logistic family keeps the update on the probability scale; `.coef` extracts the single ε; a small number close to zero means the initial Q needed little correction

---

# Counterfactual Predictions for TMLE

```{webr-r}
# Get initial (pre-fluctuation) predictions under each treatment
Q1_init <- predict(Qmod, newdata = mutate(dat, A=1), type="response")
Q0_init <- predict(Qmod, newdata = mutate(dat, A=0), type="response")

# Clever covariates for counterfactual treatment values
H1 <- 1 / ps          # <1>
H0 <- -1 / (1 - ps)   # <1>

# Apply the fluctuation to counterfactual predictions
Q1_star <- plogis(logit(Q1_init) + epsilon * H1)  # <2>
Q0_star <- plogis(logit(Q0_init) + epsilon * H0)  # <2>

# TMLE ATE estimate
tmle_ate <- mean(Q1_star) - mean(Q0_star)  # <3>
tmle_ate
```
1. Treatment-arm-specific clever covariates: $H_1(W) = 1/g(W)$ targets the treated potential outcome; $H_0(W) = -1/(1-g(W))$ targets the control potential outcome. These must be evaluated at the *counterfactual* treatment value, not the observed one.
2. Apply the fluctuation on the logit scale, then `plogis()` maps back to probability; guaranteeing all targeted predictions stay in $[0,1]$, unlike AIPW which can violate these bounds
3. The TMLE **substitution estimator**: simply take the sample mean of each arm's targeted predictions. No explicit weighting needed; the clever covariate already incorporated the propensity score.

The initial outcome and propensity score models do not have to be simple GLMs. They can be estimated with Super Learner, an ensemble machine learning approach covered in the next chapter, which reduces sensitivity to model misspecification.

# TMLE Inference: Standard Errors and Confidence Intervals

A major advantage of TMLE is that inference follows directly from the **efficient influence curve (EIC)**. Using the clever covariate notation from Section 7, where $H(A, W) = A/g(W) - (1-A)/(1-g(W))$, the EIC for the ATE at the TMLE fit can be written in a single unified form:

$$
D_i = H(A_i, W_i)\big(Y_i - Q^*(A_i, W_i)\big) + Q^*(1, W_i) - Q^*(0, W_i) - \hat{\psi}
$$

Equivalently, expanding $H$ into its treatment-arm-specific components $H_1(W) = 1/g(W)$ and $H_0(W) = -1/(1-g(W))$ (which already include the sign for each arm):

$$
D_i = \frac{A_i}{e(W_i)}\big(Y_i - Q_1^*(W_i)\big) - \frac{1-A_i}{1-e(W_i)}\big(Y_i - Q_0^*(W_i)\big) + Q_1^*(W_i) - Q_0^*(W_i) - \hat{\psi}
$$

where $\hat{\psi}$ is the TMLE ATE estimate. The variance of the ATE is estimated by the sample variance of the EIC divided by $n$: $\hat{\sigma}^2 = \frac{1}{n}\sum_i D_i^2$.

::: {.callout-important}
## When is EIC-based inference valid?

The EIC-based variance estimator and resulting Wald-type confidence intervals are asymptotically valid when (i) at least one nuisance model is consistently estimated, (ii) positivity holds, and (iii) certain regularity/rate conditions are met. When **data-adaptive or machine learning** methods are used for nuisance estimation, these conditions require **cross-fitting** (sample-splitting) or **CV-TMLE** to avoid empirical process conditions (Donsker class membership) that flexible learners typically violate. In the `tmle3` and `drtmle` R packages, cross-fitting is available; the classic `tmle` package does not cross-fit by default but remains valid when the Super Learner library is not too aggressive (e.g., excludes highly adaptive learners like HAL without cross-fitting). See @zheng2011crossvalidated for the theoretical foundations.
:::

```{webr-r}
# Observed outcomes
Y_obs <- dat$Y

# EIC (efficient influence curve) components
# For treated: use Q1_star; for control: use Q0_star
# But we need Q* at the OBSERVED A for the residual terms
Qstar_obs <- ifelse(dat$A == 1, Q1_star, Q0_star)  # <1>
H_obs <- ifelse(dat$A == 1, H1, H0)                 # <1>

# Efficient influence curve for each observation
EIC <- H_obs * (Y_obs - Qstar_obs) + Q1_star - Q0_star - tmle_ate  # <2>

# Standard error and 95% CI
tmle_se <- sqrt(var(EIC) / n)  # <3>
ci_lower <- tmle_ate - 1.96 * tmle_se
ci_upper <- tmle_ate + 1.96 * tmle_se

tibble(
  Estimate = tmle_ate,
  SE = tmle_se,
  CI_lower = ci_lower,
  CI_upper = ci_upper
)
```
1. Select the targeted Q prediction and clever covariate at each observation's *observed* treatment value; needed to form the IPW-weighted residual term that corrects for outcome model error
2. The EIC has two parts: (a) $H(A,W)(Y - Q^*)$; an IPW-weighted residual that corrects any outcome model bias, and (b) $(Q_1^* - Q_0^* - \hat\psi)$; the centered substitution component. When the fluctuation step correctly solved the EIC equation, the sample mean of EIC is exactly zero.
3. The EIC variance divided by $n$ yields the semiparametric efficient variance; this is valid even when machine learning is used for nuisance models (with appropriate cross-fitting)

:::{.callout-tip}
## Interpreting TMLE Inference

The EIC-based standard error is a key output of TMLE. Because the fluctuation step solves the EIC estimating equation, the resulting confidence interval achieves approximately correct coverage under regularity conditions, even when machine learning is used for the nuisance models (provided cross-fitting or other sample-splitting techniques are employed).

If the confidence interval excludes zero, we have evidence that the treatment has a non-null causal effect on the outcome at the 95% confidence level.
:::

# Comparison: AIPW vs TMLE

### AIPW vs TMLE: Head-to-Head

| Property | AIPW | TMLE |
|---|---|---|
| Doubly robust | Yes | Yes |
| Semiparametrically efficient | Yes (asymptotically) | Yes |
| Handles ML well | Yes, with cross-fitting | Yes, with SuperLearner + cross-fitting (CV-TMLE) |
| Predictions bounded in $[0,1]$ | No | Yes |
| Solves EIC equation | Not guaranteed | Yes, by construction |
| Implementation complexity | Simple | Moderate |

**Key point.** Both AIPW and TMLE are valid doubly robust estimators. TMLE is generally preferred because it respects the bounds of the parameter space (e.g., probabilities stay in $[0,1]$), integrates naturally with ensemble machine learning, and solves the efficient influence curve equation by construction. In practice, this means TMLE tends to have better finite-sample performance and more trustworthy inference.

## Check your understanding

<details>
<summary><strong>Q1. What does "doubly robust" mean in practice? Does it mean you can ignore model specification entirely?</strong></summary>

No. "Doubly robust" means the estimator is consistent if *either* the treatment model or the outcome model is correctly specified, but not necessarily if both are wrong. It does not give you license to ignore model specification. If both nuisance models are badly misspecified, the estimator can be biased. In practice, you should still try to specify both models as carefully as possible. The double robustness property is an insurance policy, not a substitute for thoughtful modeling.

</details>

<details>
<summary><strong>Q2. Why does TMLE use a "clever covariate" derived from the propensity score, and what happens if propensity scores are extreme?</strong></summary>

The clever covariate $H(A, W) = A / g(W) - (1 - A) / (1 - g(W))$ is derived from the efficient influence curve for the ATE. It directs the fluctuation of the initial outcome model toward the specific causal parameter of interest, ensuring that the final estimate solves the EIC estimating equation. If propensity scores are extreme (near 0 or 1), the clever covariate becomes very large, which can destabilize the fluctuation step. This is why practical positivity (adequate overlap in propensity score distributions) is important for TMLE just as it is for IPTW.

</details>

<details>
<summary><strong>Q3. Your TMLE estimate of a risk probability is 0.85. Could AIPW have produced an estimate above 1.0 for the same data?</strong></summary>

Yes. AIPW is a linear estimator that does not enforce the bounds of the parameter space. Because it adds a weighted residual correction to the outcome-model prediction, the resulting estimate can fall outside $[0, 1]$, especially when propensity scores are extreme and the correction term is large. TMLE avoids this by performing its update on the logistic (log-odds) scale and then back-transforming, which guarantees that predicted probabilities remain in $[0, 1]$. This is one of the key practical advantages of TMLE over AIPW.

</details>

## Software Implementation (R)

This example demonstrates the **double robustness** property using the `tmle` package. We deliberately misspecify one nuisance model at a time to show that the TMLE estimate remains consistent as long as at least one model is correct.

- Simulate data with a nonlinear confounder–outcome relationship
- Scenario A: correct $Q$, wrong $g$ → TMLE consistent
- Scenario B: wrong $Q$, correct $g$ → TMLE consistent
- Scenario C: both correct → TMLE efficient
- Falls back to a manual demonstration if `tmle` is unavailable

```{r}
#| eval: false
set.seed(1)
n <- 1000
W <- rnorm(n)
A <- rbinom(n, 1, plogis(0.3 * W + 0.2 * W^2))
Y <- 0.5 * A + sin(W) + rnorm(n, sd = 0.3)  # true ATE = 0.5

if (requireNamespace("tmle", quietly = TRUE)) {
  library(tmle)

  ## Scenario A: correct Q model, misspecified g (intercept only)
  fit_a <- tmle(Y = Y, A = A, W = data.frame(W = W),  # <1>
                Q.SL.library = c("SL.glm", "SL.earth"),  # <2>
                g.SL.library  = "SL.mean")               # <3>
  cat("Scenario A (correct Q, wrong g):", round(fit_a$estimates$ATE$psi, 3), "\n")

  ## Scenario B: misspecified Q (intercept only), correct g
  fit_b <- tmle(Y = Y, A = A, W = data.frame(W = W),
                Q.SL.library = "SL.mean",                # <4>
                g.SL.library  = c("SL.glm", "SL.earth"))
  cat("Scenario B (wrong Q, correct g):", round(fit_b$estimates$ATE$psi, 3), "\n")

  ## Scenario C: both reasonably specified
  fit_c <- tmle(Y = Y, A = A, W = data.frame(W = W),
                Q.SL.library = c("SL.glm", "SL.earth"),
                g.SL.library  = c("SL.glm", "SL.earth"))
  cat("Scenario C (both correct):      ", round(fit_c$estimates$ATE$psi, 3), "\n")
  cat("95% CI:", round(fit_c$estimates$ATE$CI, 3), "\n")
} else {
  message("Install the 'tmle' package:  install.packages('tmle')")
}
```
1. `tmle()` from the `tmle` package automates all four TMLE steps; `W` must be a `data.frame` (not a matrix or bare vector)
2. `Q.SL.library`: the Super Learner library for the outcome model $E[Y \mid A, W]$; `SL.earth` adds MARS splines to capture nonlinearity in the outcome
3. `g.SL.library = "SL.mean"`: deliberately misspecified propensity score using only an intercept-only model (predicts the marginal mean treatment probability for everyone); this tests whether TMLE remains consistent when Q is correct but g is wrong
4. `Q.SL.library = "SL.mean"`: deliberately misspecified outcome model; tests whether TMLE remains consistent when g is correct but Q is wrong; both Scenarios A and B should give estimates close to the true ATE = 0.5
